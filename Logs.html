Trying to Git gud

A meta

I'm using Github pages to bring this site up, hence, I needed to create a repo with all the site content. I haven't used Git before, only SVN. They have some very fundamental differences, one being that there is more checks and balances with Git as opposed to SVN (or at least in the SVN environment I used to work at). I can now see why my interviewer was not amused when I was describing the workflow at my previous job. PSA to my future self: Local working copy to repo is a no-no, even if that was how it was done where you worked at.

Back to the topic at hand, Git workflow goes like this:

  1. Clone the repo: git clone repo_url
  2. Create a branch for whichever feature you want to work on: git checkout -b branch_name
  3. Commit changes in your local repository: git commit -am "your message here". As a sidenote, this is where SVN and Git really diverges. A commit in SVN means that it is uploaded to the main repository (i.e. not local) whereas a Git commit is like logging the changes on the local repository you have cloned, specifically on the branch you have checked out.
  4. Push the changes to your Github repo: git push origin branch_name. Another sidenote, this is when it becomes available remotely.
  5. Something about pull requests and yada yada that isn't really relevant to what I'm trying to do right now.

Since I'm a lunatic, I'm probably not going to create separate branches everytime I update the site. Therefore I will only be using the main branch. Note to self, the command is "git push origin main".

Git also complains when you try to commit and haven't identified yourself (picture medieval knights). The commands to set them are:

git config --global user.email "your_email@server.com"

git config --global user.name "kimi_no_nawa"

Also due to myself not wanting to advertise my email publicly everytime I make a commit, I have to use the email address available at my Github account settings page under the Email tab. If I use my email address, git will throw an error saying this commit will show my email address which is not allowed because I told Github to keep my email address private so yada yada either change it or untick that option.

Alright so there I have it, the basics of Git. Good enough to put in my resume perhaps.
